The toggle component creates an input for users to toggle between two states. The change takes effect immediately.

Use when

Use toggles when users can toggle between two states or options, and the changes take effect immediately.
Do use toggles to alternate between two states when changes take effect immediately.
Do use toggles to set binary states on multiple items.

Don't use when

Don’t use toggles when the options require confirmation before changes take effect.
Don't use toggles for states with significant consequences. Use a workflow where users must confirm the changes instead.

Don’t use toggles when the input is presented in a modal.
Don't use toggles inside modals where they won't take effect immediately. Use checkboxes instead.
 Toggle body Text label

Text label

When toggles are in grid columns, the text label is not required. Use text labels for all other states.

If the scenario allows for toggles to become disabled, always include a text label on each toggle.
